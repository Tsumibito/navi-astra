# Features — Navi Astro (Astro 5 + Tailwind)

Этот документ описывает реализованные фичи фронтенда в `navi-astro/`: какие задачи решают, где находятся в коде, какие данные используют/отдают, как тестировать и как переиспользовать.

## Обзор
- Стек: Astro 5 (SSG) + Tailwind CSS.
- Деплой: Cloudflare Pages (см. `README.md`).
- Статические ассеты: `public/` (включая `.avif` изображения).
- Кодовая база:
  - Страницы: `src/pages/`
  - Лэйауты: `src/layouts/`
  - Компоненты: `src/components/`
  - Стили: `src/styles/global.css`
  - Конфиги: `astro.config.mjs`, `tailwind.config.cjs`, `postcss.config.cjs`, `tsconfig.json`, `package.json`

## Фичи

### 1) Базовый лэйаут с i18n-навигацией и мобильным сайдбаром
- Где: `src/layouts/BaseLayout.astro`
- Что делает:
  - Оборачивает контент страниц (`<slot />`) общим каркасом: `<Header />`, `<Footer />`, full-bleed `<PhotoStrip />` и контейнер `main`.
  - Принимает пропсы: `title`, `description`, `locale` (по умолчанию `ru`).
  - Формирует меню по локали через локальные мапы (`navByLocale`) и отрисовывает мобильный выезжающий сайдбар (off-canvas) со списком ссылок, переключателем языков и кнопкой CTA.
- Данные/отдаёт:
  - Внутренние данные: `navByLocale` и список `langs` (`['ru','ua','en']`).
  - Передаёт `locale` вниз в компоненты.
- Как тестировать:
  1. `npm run dev` и открыть `/`.
  2. Проверить хедер и мобильное меню: при клике по бургеру появляется сайдбар; backdrop корректно управляет прокруткой; переключение иконки (бургер/крестик).
  3. Проверить ссылки в меню для `ru`, `ua`, `en`, `fr` (передать `locale` при использовании `BaseLayout`).
- Переиспользование:
  - Используйте `BaseLayout` как оболочку для новых страниц, прокидывайте `locale`, `title`, `description`.

### 2) Хедер с бургером, desktop-lang dropdown и доступностью
- Где: `src/components/Header.astro`
- Что делает:
  - Рендерит логотип, центральное меню (десктоп), языковой дропдаун (десктоп) и бургер (мобайл).
  - JS-логика без фреймворков: управление aria-атрибутами, плавные анимации для сайдбара и backdrop, резайз-обработка высоты хедера, защита от горизонтального скролла.
- Как тестировать:
  - Проверить клавиатурную навигацию и aria `aria-expanded` на кнопке бургера; посмотреть, что backdrop и drawer корректно показываются/скрываются и блокируют прокрутку.
- Переиспользование:
  - При расширении меню/языков добавляйте элементы в локальные структуры `navByLocale` и `langs`.

### 3) Меню навигации (универсальный компонент)
- Где: `src/components/NavMenu.astro`
- Что делает: отрисовывает список ссылок с кастомным классом.
- Данные: пропсы `{ items: {label, href}[], class?: string }`.
- Тестирование: визуально на десктопе и мобайле (в сайдбаре).
- Переиспользование: используйте в любом месте, передавая массив ссылок.

### 4) Фото-лента (full-bleed) с .avif
- Где: `src/components/PhotoStrip.astro`
- Что делает: рендерит ряд изображений, растянутый на ширину вьюпорта (full-bleed), с повторением набора до 24 элементов.
- Данные: проп `images?: Array<{ src: string; alt?: string; href?: string }>`; по умолчанию использует `public/photos/1..12.avif`.
- Тестирование: проверить на разных размерах экрана, что изображения тянутся и скролла по X нет.
- Переиспользование: можно передавать свой массив картинок.
- Примечание: сейчас состоят из `<img src>`. Для автоматической оптимизации и responsive-выводов можно внедрить Astro Images (см. TODO ниже).

### 5) Футер с локализуемыми колонками и соц. иконками
- Где: `src/components/Footer.astro`
- Что делает: логотип, две локализуемые колонки ссылок (по `locale`), блок соц. иконок (Telegram, Facebook, Instagram, LinkedIn, YouTube), копирайт и policy-ссылки.
- Тестирование: проверить корректность ссылок и адаптивную сетку.
- Переиспользование: при расширении — добавляйте пункты в структуры `links` по локалям.

### 6) Главная страница (демо)
- Где: `src/pages/index.astro`
- Что делает: пример использования `BaseLayout`, карточки-ссылки на документации Astro и Tailwind.
- Переиспользование: заменить контент на реальный, сохранив оболочку лэйаута.

## Стили и тема
- Общие стили: `src/styles/global.css` — Tailwind directives и CSS-переменные.
- Tailwind: `tailwind.config.cjs` — кастомные цвета бренда (`brand-orange`, `brand-orangeDark`, `brand-blue`), семейства шрифтов `neo` и `tenor`.
- PostCSS: `postcss.config.cjs` — `tailwindcss` + `autoprefixer`.

## Конфигурация и сборка
- `astro.config.mjs`: интеграция `@astrojs/tailwind` с `applyBaseStyles: true`.
- `package.json` скрипты: `dev`, `build`, `preview`, `check`.
- Деплой на Cloudflare Pages: см. `README.md` (Build: `npm run build`, Output: `dist/`, Node 18+).

## Тестирование (быстрый чек-лист)
1. Установить зависимости: `npm i` в папке `navi-astro/`.
2. Запустить: `npm run dev` и открыть локальный адрес (обычно `http://localhost:4321`).
3. Проверить хедер/бургер/сайдбар, переключение языков (desktop dropdown), full-bleed фотоленту.
4. Прогнать Lighthouse: ожидается высокий балл благодаря SSG и минимальному JS.

## Переиспользование
- `BaseLayout` — для всех страниц (передавайте `locale`, `title`, `description`).
- `NavMenu` — универсален для любых наборов ссылок.
- `PhotoStrip` — можно вставлять на лендинги, теги, посты с кастомным списком изображений.
- `Footer` — расширяется локализуемыми ссылками.

## Планируемая связка с Sanity (SSG, без рантайм‑запросов)
Цель: полностью статическая генерация (SSG) без запросов к API в рантайме. Все данные подтягиваются на этапе билда и кладутся в репозиторий проекта или артефакты билда.

__Вариант A — Astro Content Collections__
- Структура:
  - `src/content/config.ts` — описание коллекций (`posts`, `tags`, `team`, `certificates`, `faqs`, `redirects`, `globals`).
  - `src/content/posts/*.json` (или `.mdx` при необходимости MDX) — документы с валидированными типами.
  - Аналогично для `tags`, `team`, `certificates`, `faqs`.
- Импорт: `const entries = await getCollection('posts')` прямо в страницах/роутах.
- Генерация данных: отдельный скрипт, который тянет из Sanity (GROQ) и конвертирует к схемам Content Collections.

__Вариант B — Статические JSON__
- Структура:
  - `src/data/posts.json`
  - `src/data/tags.json`
  - `src/data/team.json`
  - `src/data/certificates.json`
  - `src/data/faqs.json`
  - `src/data/redirects.json`
  - `src/data/globals.json`
- Импорт: обычный `import posts from "@/data/posts.json"` в компонентах/страницах.
- Плюсы: проще мигрировать и кэшировать, меньше связки с Astro API.

__Скрипт импорта (build-time)__
- Файл: `scripts/fetch-content.mjs`.
- Действия:
  1. Читает `.env` (SANITY_PROJECT_ID, SANITY_DATASET, опционально токен с `read`-ролями, если требуется доступ к черновикам).
  2. Выполняет GROQ-запросы к Sanity для `posts`, `tags`, `team`, `certificates`, `faqs`, `redirects`, `globals`.
  3. Преобразует поля к фронтовой форме (например: `seo` объект, локализованные `slug`/`name`, ссылки на изображения).
  4. Сохраняет файлы в `src/content/**` (Вариант A) или `src/data/*.json` (Вариант B).
- Интеграция в сборку:
  - Добавить npm-скрипт: `"prebuild": "node scripts/fetch-content.mjs"`.
  - Затем обычный `astro build` работает уже на локальных данных, без сети.

__Изображения__
- Источники: Sanity CDN или Cloudflare R2 (в зависимости от итогового решения бэкенда).
- Рендеринг: внедрить Astro Images (или `<Image />` Astro 5) и прокинуть `srcSet/sizes` для десктоп/планшет/мобильный.
- Рекомендации:
  - Для OG — 1200×630 (1.91:1).
  - Для карточек — предусмотреть несколько брейкпоинтов (напр., 320, 640, 960, 1280) и форматы `webp/avif`.

__Маршруты и i18n__
- Генерировать страницы под локали `ru/ua/en` (и `fr`, если используется) из одного источника данных.
- Cобрать карту slug’ов на язык и сформировать пути: `/ru/blog/[slug]`, `/ua/blog/[slug]`, `/en/blog/[slug]` и т.д.
- Редиректы: читать `redirects.json` и сформировать `_redirects` (Cloudflare Pages) или применить на уровне Astro маршрутизации (статические файлы).

__Тестирование__
- Запуск: `npm run prebuild && npm run build && npm run preview`.
- Проверить корректность локалей, наличие всех страниц и соответствие контента ожидаемому.
- Lighthouse и WebPageTest — для оценки перформанса, особенно изображений и критического рендера.
